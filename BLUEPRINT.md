# 项目蓝图 — Citrinitas

> 这是 Citrinitas 的项目宪法。只写方向和原则，不写技术细节。
> 具体怎么做，看 PROJECT_PLAN.md。
>
> **Citrinitas 是 OpusMagnum（巨作）一人公司蓝图的一环**：
> Nigredo(视频→知识) → Citrinitas(知识引擎) → Albedo(矛盾检测) → Rubedo(发布)

## 愿景

让管理个人知识像和人聊天一样自然。

把"翻文件"变成"问知识"——拖进去，搜索时 AI 帮你找到并回答。

**与通用 AI 的区别**：通用 LLM 为所有人服务，Citrinitas 深度为你服务。基础知识库保证准确，不胡说；个人知识库高度适配你的习惯。

**长久陪伴**：管理日常闪现的灵感不被遗忘。桌面端跑通 MVP 后，走向手机随身操作。

## 核心原则

1. **入口无感** — 用户不应该操心知识从哪来。拖文件、网页剪藏、上游程序推送……都能自动识别入库。
2. **结果可信** — 每次回答都能追溯到出处。不说胡话，不编造。分类不准确时用户可以修正。
3. **私人适配** — 越用越懂你。不是你适应工具，是工具适应你。
4. **工序衔接** — Citrinitas 是巨作流水线的一环。上游程序产出的知识自动流入，下游程序可以调用 Citrinitas 基于你的习惯来研究知识。最终：程序替你研究，你验收结果。

## 当前重心

**让知识"进得来、找得到、看得见"** —— 降低摄入门槛，提升搜索质量，建立知识之间的关联。

1. **进得来** — 入口顺畅，用户愿意往里放东西（摄入体验）
2. **找得到** — 搜索准确，用户能找到想要的知识（搜索优化）
3. **看得见** — 知识关联可视化，用户理解知识网络（图谱 + 审核队列）

> 详细的数据流程和节点定义见 [FLOWCHART.md](./FLOWCHART.md)。

## 边界

- 不做多人协作（个人工具）
- 不做云端同步
- 不做移动端（首版桌面跑通 MVP）
- 不做插件系统（v1.0 之后的事）
- 不做通用 API 平台（开放接口只服务于"私人知识引擎"这个目标）

## 验收标准

- 普通人不需要说明书，凭直觉就能完成"拖入文件 → 找到内容"
- 摄入失败时不静默——用户知道发生了什么
- 搜索出来的东西有出处——能追溯到原始文件
